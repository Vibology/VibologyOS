## 2. Analysis Criteria

### Required Subsections
Files were evaluated for presence of these essential subsections:

1. **Mythology and Cross-Tradition Parallels** (with 3-5 examples from different traditions)
2. **Relationships with Other Archangels** (referencing 3-4 other archangels)
3. **Cross-System Correspondences** with subsections for:
   - Qabalah
   - Tarot
   - Astrology
   - Human Design (or other relevant system)
   - Jungian Psychology

### Stub Sections
Content sections containing fewer than 5 lines of actual text (excluding headers and blank lines).

### Duplicate Headers
Instances where the same section header appears multiple times in a file.

### Depth Consistency
Comparative analysis of standard subsections across files to identify variations in treatment depth.

## 12. Recommendations for Standardization

### Priority 1: Critical Issues

1. **Raphael.md File Size**
   - File exceeds analysis token limit (28,096 tokens)
   - **Action:** Manually review for:
     - Duplicate "Qabalistic Context" sections (user-reported issue)
     - Potential content that could be moved to sub-files
     - Redundancy that could be condensed
   - **Alternative:** Read in chunks using offset/limit to conduct analysis

### Priority 2: Consistency Enhancements

2. **I-Ching Integration**
   - Currently present in Michael.md (Hexagram 1)
   - Absent or not explicit in Gabriel, Uriel, Metatron
   - **Action:** Establish policy:
     - Option A: Add I-Ching correspondences to all files where applicable
     - Option B: Mark I-Ching as optional/contextual rather than required
     - Option C: Integrate via "The Window" system references

3. **Cross-System Correspondences Section Structure**
   - Some files integrate correspondences throughout (Uriel)
   - Others have dedicated section with subsections (Michael, Gabriel)
   - **Action:** Standardize whether Cross-System section should:
     - Be comprehensive standalone section, OR
     - Serve as summary with references to detailed sections earlier in file

4. **Verification Notes Standardization**
   - Recent files (2026-01-25) have excellent verification
   - **Action:** Ensure all files include:
     - `verified: true/false` in YAML
     - `verification_source` field
     - `verification_notes` documenting what was checked/removed/flagged

### Priority 3: Enhancement Opportunities

5. **Relationships Section Depth**
   - Consider standardizing whether to include:
     - Just descriptive relationships, OR
     - Functional dynamics between archangels, OR
     - Both
   - Michael and Metatron demonstrate excellent relational dynamics discussion

6. **Fallen Aspect Section**
   - All analyzed files include 4-5 distinct shadow manifestations
   - Consider whether this is appropriate standard for all files
   - Ensure each shadow aspect includes:
     - Manifestation description
     - Examples
     - Pathology identification
     - Correction/antidote

7. **Invocation Section Structure**
   - Files vary in approach:
     - Gabriel: Meditation + prayers + ritual (LBRP)
     - Michael: When to invoke + traditional prayers + LBRP + meditation
     - Uriel: Embodiment practices + descent work + LBRP + contemplation
     - Metatron: Meditation pathworking + invocation methods + signs
   - **Action:** Identify optimal structure that includes:
     - When/why to invoke
     - Traditional invocation texts
     - Practical methods/rituals
     - Signs of presence
     - Warnings/cautions
